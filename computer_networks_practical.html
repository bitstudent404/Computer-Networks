Basic Network & Network Configuration Commands

💻 Part A — Basic Network Commands (Windows CMD)
CommandPurpose
ipconfigDisplays IP address, Subnet Mask, Default Gateway
ipconfig /allFull details including MAC address, DHCP status
ping 192.168.1.1Tests connectivity to a target device
tracert 8.8.8.8Traces the route packets take to a destination
netstatDisplays all active connections and ports
nslookup google.comDNS lookup — resolves hostname to IP
arp -aShows ARP table (IP-to-MAC address mapping)
hostnameShows the name of the current computer
netstat -anShows all ports listening and established
net viewLists devices on the local network
⚙️ Part B — Cisco Router/Switch CLI Commands
Cisco IOS CLI 
# Enter Privileged EXEC Mode
Router> enable

# Enter Global Configuration Mode
Router# configure terminal

# Set Hostname
Router(config)# hostname R1

# Configure an Interface
R1(config)# interface fastEthernet 0/0
R1(config-if)# ip address 192.168.1.1 255.255.255.0
R1(config-if)# no shutdown
R1(config-if)# exit

# Useful Show Commands
R1# show ip interface brief       ← View all interfaces + status
R1# show running-config           ← View full current config
R1# show version                  ← Hardware and IOS version
R1# show ip route                 ← View routing table

# Save Configuration
R1# write memory

Practical 2

PC to PC Communication

📌
Two PCs connected directly to each other (without a switch) require a Cross-Over Cable. You are creating a direct peer-to-peer network.
🪜 Step-by-Step in Cisco Packet Tracer
  • 1

    Open Packet Tracer → Click End Devices panel → Drag 2 PCs onto the workspace.

  • 2

    Click Connections (lightning bolt icon) → Select Copper Cross-Over Cable.

  • 3

    Click PC0 → choose FastEthernet0, then click PC1 → choose FastEthernet0. A green link light confirms connection.

  • 4

    Click PC0 → Desktop tab → IP Configuration:
    IP Address: 192.168.1.1  |  Subnet Mask: 255.255.255.0

  • 5

    Click PC1 → same path:
    IP Address: 192.168.1.2  |  Subnet Mask: 255.255.255.0

  • 6

    Click PC0 → Desktop → Command Prompt → Type the ping command below.

PC0 — Command Prompt 
PC> ping 192.168.1.2

Reply from 192.168.1.2: bytes=32 time<1ms TTL=128
Reply from 192.168.1.2: bytes=32 time<1ms TTL=128
Reply from 192.168.1.2: bytes=32 time<1ms TTL=128
Reply from 192.168.1.2: bytes=32 time<1ms TTL=128
✅ Communication Successful!

Practical 3

Star Topology using Hub and Switch

⭐ Star Topology with HUB
  • 1

    Add 4 PCs + 1 Hub (from Network Devices → Hubs) to the workspace.

  • 2

    Connect each PC to the Hub using a Straight-Through Cable (PC→Hub always uses straight-through).

  • 3

    Assign IPs: PC0=192.168.1.1, PC1=192.168.1.2, PC2=192.168.1.3, PC3=192.168.1.4 (same subnet mask 255.255.255.0).

  • 4

    Ping between any two PCs to verify. All should reply ✅

🔀 Star Topology with SWITCH
  • 1

    Replace Hub with a Switch (Network Devices → Switches → 2960). All other steps are identical.

  • 2

    Connect each PC to the Switch using Straight-Through Cable.

  • 3

    Assign the same IPs as above and ping to test.

💡
Hub vs Switch: A Hub broadcasts data to ALL connected PCs (inefficient). A Switch learns MAC addresses and sends data ONLY to the intended PC — far more efficient and secure.

Practical 4

Bus, Ring, Tree, Hybrid, Mesh Topologies

🔵 Bus

PCs in a line sharing one backbone cable. Use a Hub in a line or connect PCs through repeaters linearly.

🔴 Ring

Each PC connects to the next forming a loop. PC1→PC2→PC3→PC4→back to PC1.

🟢 Tree

Root switch connects to sub-switches, each with PCs attached. Hierarchical star.

🟡 Hybrid

Mix of two topologies (e.g., two Star networks connected via a Bus link between switches).

🟣 Mesh

Every router connects to every other. 4 nodes = 6 connections. Formula: n×(n-1)/2.

🪜 How to Build Each in Packet Tracer

🔵 Bus Topology:

  • 1

    Add 4 PCs + 1 Hub. Connect PC1→Hub1, PC2→Hub1, Hub1→Hub2, PC3→Hub2, PC4→Hub2 in a linear chain.

  • 2

    Assign IPs in the same subnet. Ping to test.

🔴 Ring Topology:

  • 1

    Add 4 PCs. Use Cross-Over Cables to connect PC0→PC1→PC2→PC3→back to PC0.

  • 2

    Assign IPs. Note: In Packet Tracer, each PC has one NIC so ring is simulated logically.

🟢 Tree Topology:

  • 1

    Add 1 Root Switch + 2 Sub-Switches. Connect Root→Sub1 and Root→Sub2 with Straight-Through cables.

  • 2

    Add 2-3 PCs to each sub-switch. Assign IPs in same subnet.

🟣 Mesh Topology:

  • 1

    Add 4 Routers. Connect R1↔R2, R1↔R3, R1↔R4, R2↔R3, R2↔R4, R3↔R4 (6 total connections).

  • 2

    Assign a unique IP to each interface. Configure routes as needed.

Practical 5

Initial Switch Configuration

📌
Click the Switch → CLI tab in Packet Tracer. Press Enter to begin, then type the commands below.
Switch CLI — Initial Configuration 
# Enter privileged mode
Switch> enable
Switch# configure terminal

# Set hostname
Switch(config)# hostname SW1

# Set enable password (to protect privileged mode)
SW1(config)# enable password cisco123

# Set console line password
SW1(config)# line console 0
SW1(config-line)# password console123
SW1(config-line)# login
SW1(config-line)# exit

# Set VTY (Telnet) password
SW1(config)# line vty 0 4
SW1(config-line)# password telnet123
SW1(config-line)# login
SW1(config-line)# exit

# Set login banner message
SW1(config)# banner motd #Authorized Access Only!#

# Assign management IP to VLAN 1 interface
SW1(config)# interface vlan 1
SW1(config-if)# ip address 192.168.1.2 255.255.255.0
SW1(config-if)# no shutdown
SW1(config-if)# exit

# Disable unused ports for security
SW1(config)# interface range fa0/5 - 24
SW1(config-if-range)# shutdown
SW1(config-if-range)# exit

# Save configuration
SW1# write memory
SW1# show running-config        ← Verify all settings

Initial Router Configuration

Router CLI — Initial Configuration 
# Enter privileged and config mode
Router> enable
Router# configure terminal

# Set hostname
Router(config)# hostname R1

# Set encrypted enable password
R1(config)# enable secret mypassword

# Configure LAN-facing interface (Gig0/0)
R1(config)# interface GigabitEthernet0/0
R1(config-if)# ip address 192.168.1.1 255.255.255.0
R1(config-if)# no shutdown
R1(config-if)# exit

# Configure WAN-facing interface (Gig0/1)
R1(config)# interface GigabitEthernet0/1
R1(config-if)# ip address 10.0.0.1 255.255.255.0
R1(config-if)# no shutdown
R1(config-if)# exit

# Console password
R1(config)# line console 0
R1(config-line)# password routerpass
R1(config-line)# login
R1(config-line)# exit

# Telnet (VTY) password
R1(config)# line vty 0 4
R1(config-line)# password vtypass
R1(config-line)# login
R1(config-line)# exit

# Save config
R1# write memory
R1# show ip interface brief       ← Both interfaces should show "up"

Practical 7

Client-Server Network

🌐 Network Layout
📌
1 Server + 3 PCs (clients) + 1 Switch. Server hosts HTTP website + DHCP to auto-assign IPs to clients.
  • 1

    Add 1 Server (End Devices → Server), 3 PCs, and 1 Switch. Connect all devices to the Switch using Straight-Through cables.

  • 2

    Click Server → Desktop → IP Configuration → Set manually:
    IP: 192.168.1.100 | Mask: 255.255.255.0 | Gateway: 192.168.1.100

  • 3

    Click Server → Services tab → DHCP → Toggle to ON. Set: Default Gateway = 192.168.1.100, Start IP = 192.168.1.1, Max Users = 50. Click Save.

  • 4

    Also in Services → HTTP → Toggle to ON. The default webpage is already set.

  • 5

    On each PC → Desktop → IP Configuration → Select DHCP. The PC will automatically receive an IP from the server ✅

  • 6

    On any PC → Desktop → Web Browser → Type 192.168.1.100 in address bar → The server's default webpage loads ✅

Practical 8

Connection Between Devices Using a Router

📌
Two separate networks (192.168.1.0/24 and 192.168.2.0/24) connected through one Router. The router acts as the gateway between them.
🪜 Steps in Packet Tracer
  • 1

    Add: 2 PCs, 2 Switches, 1 Router (Routers → 2911).

  • 2

    Connect: PC0 → Switch0 → Router (Gig0/0) using Straight-Through cables.

  • 3

    Connect: PC1 → Switch1 → Router (Gig0/1) using Straight-Through cables.

  • 4

    Configure the Router CLI (see code below).

  • 5

    PC0 config: IP=192.168.1.2 | Mask=255.255.255.0 | Gateway=192.168.1.1

  • 6

    PC1 config: IP=192.168.2.2 | Mask=255.255.255.0 | Gateway=192.168.2.1

  • 7

    From PC0 → Command Prompt → ping 192.168.2.2 → ✅ Replies confirm inter-network routing works!

Router CLI — Inter-Network Routing 
# Interface towards Network 1
Router(config)# interface GigabitEthernet0/0
Router(config-if)# ip address 192.168.1.1 255.255.255.0
Router(config-if)# no shutdown
Router(config-if)# exit

# Interface towards Network 2
Router(config)# interface GigabitEthernet0/1
Router(config-if)# ip address 192.168.2.1 255.255.255.0
Router(config-if)# no shutdown
Router(config-if)# exit

# Save and verify
Router# write memory
Router# show ip route    ← Should show both C 192.168.1.0 and C 192.168.2.0

Practical 9

Remote Desktop Sharing within LAN Connection

⚠️
Packet Tracer does not support actual RDP (port 3389). Telnet is used to simulate remote access in Packet Tracer. For real-world RDP steps, see Part B below.
🖥 Part A — Telnet Remote Access (Packet Tracer Simulation)
  • 1

    Build a LAN: 2 PCs connected to a Switch. Assign IPs: PC0=192.168.1.10, PC1=192.168.1.20, Mask=255.255.255.0.

  • 2

    Add a Router or Switch to serve as the remote device. Configure Telnet on it (see CLI below).

  • 3

    From PC0 → Desktop → Command Prompt → Type: telnet 192.168.1.20

  • 4

    Enter the password when prompted → You are now remotely logged in ✅

Switch/Router CLI — Enable Telnet 
# Set management IP
Switch(config)# interface vlan 1
Switch(config-if)# ip address 192.168.1.20 255.255.255.0
Switch(config-if)# no shutdown
Switch(config-if)# exit

# Enable Telnet on VTY lines
Switch(config)# line vty 0 4
Switch(config-line)# password telnetpass
Switch(config-line)# login
Switch(config-line)# exit

Switch(config)# enable password admin
Switch# write memory

# From PC0 Command Prompt:
PC> telnet 192.168.1.20
Password: telnetpass
SW1>    ← Remote access successful!
🪟 Part B — Real Windows Remote Desktop (RDP) Steps
  • 1

    On the target PC: Right-click This PC → Properties → Remote settings → Enable "Allow remote connections".

  • 2

    Note the target PC's IP address using ipconfig in CMD.

  • 3

    On the source PC: Press Win + R → type mstsc → Enter.

  • 4

    In the Remote Desktop window → Enter the target PC's IP address → Click Connect.

  • 5

    Enter the target PC's username and password → ✅ Remote desktop session opens!
